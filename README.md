##  📃Цель проекта
  Проект направлен на разработку рекомендательной системы для банка «Star». Ключевая цель — создать MVP (минимально жизнеспособный продукт), который предоставляет клиентам персонализированные предложения по банковским продуктам и услугам. Система анализирует предпочтения и поведение пользователей для оптимизации маркетинговых стратегий банка.

##  ⚙Технологический стек 
  Spring — мощный фреймворк для разработки приложений на языке Java.
  Swagger — инструмент для документирования API, который предоставляет интерфейс для визуализации.
  Spring Data JPA — часть экосистемы Spring, которая упрощает работу с базами данных.
  PostgreSQL — объектно-реляционная система управления базами данных.
  H2 — легковесная реляционная база данных.
  Liquibase — инструмент для управления версиями схемы базы данных.
  Caffeine Cache — высокопроизводительный кэш для Java-приложений.

## 💡Основные возможности
  Персонализированные рекомендации: Система осуществляет анализ клиентских данных с целью генерации индивидуализированных рекомендаций по банковским продуктам, соответствующим специфическим потребностям и предпочтениям пользователей.
  Динамическое управление правилами рекомендаций: Система предоставляет возможность внедрения и модификации правил отображения рекомендаций без необходимости в перекомпиляции и перезапуске приложения, что обеспечивает высокую гибкость и оперативность в адаптации к изменяющимся условиям и требованиям рынка.

##  📘User Stories
  Как пользователь банка, я хочу получать рекомендации по банковским продуктам на основе моих транзакций, чтобы я мог выбирать наиболее подходящие предложения.
  Как маркетолог, я хочу анализировать статистику рекомендаций, чтобы понимать, какие продукты лучше работают.
  Как аналитик, я хочу редактировать правила рекомендаций, чтобы быстро адаптировать систему к новым бизнес-требованиям.

##  ✅Установка и запуск

Предварительные требования:

- Установленная Java 23 (JDK)
  
- Установленный и запущенный сервер PostgreSQL
  
- Установленный Apache Maven
  
  Настройка базы данных
  
Перед запуском создайте базу данных в PostgreSQL для приложения

    *sql CREATE DATABASE credit_product_db;*
    
Конфигурация

  Приложение использует Spring Boot и настраивается через переменные окружения.
  
  <img width="1469" height="194" alt="image" src="https://github.com/user-attachments/assets/c4afe547-3fc0-4520-afdc-8b42c030b5aa" />

  ❗Примечание: По умолчанию проект поставляется с демонстрационной базой данных H2, которая находится в корне проекта и будет подключена автоматически, если не указана переменная
  
Сборка проекта
  Склонируйте репозиторий и выполните сборку с помощью Maven:
  *bash mvn clean package*
  
Запуск приложения

  Экспортируйте переменные окружения и запустите JAR-файл
  
*powershell env:SPRING_DATASOURCE_URL="jdbc:postgresql://localhost:5432/starbank"env:SPRING_DATASOURCE_USERNAME="your_username"         
env:SPRING_DATASOURCE_PASSWORD="your_password"env:TELEGRAM_BOT_NAME="my_star_bank_bot" $env:TELEGRAM_BOT_TOKEN="your_telegram_token"
java -jar target/starbank.jar*
